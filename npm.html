📦 Package Managers — কী, কেন এবং কোনটা ব্যবহার করবে?

JavaScript/Node.js প্রজেক্টে বাইরের library ও tool ব্যবহার করতে Package Manager অপরিহার্য। এটি ছাড়া আধুনিক web development কল্পনাই করা যায় না।

🔷 Package Manager কী?

Package Manager হলো এমন একটি টুল যা তোমার প্রজেক্টে বাইরের JavaScript library (package) ইনস্টল, আপডেট, মুছে ফেলা এবং সংস্করণ নিয়ন্ত্রণ করে। তুমি নিজে কোড লেখার পাশাপাশি অন্যদের তৈরি হাজার হাজার উপকারী library ব্যবহার করতে পারো — যেমন React, Express, Lodash, Axios ইত্যাদি। Package Manager এই সব library একটি কমান্ডে নামিয়ে দেয়।

npmjs.com হলো JavaScript-এর সবচেয়ে বড় package registry — এখানে ২০ লক্ষেরও বেশি package আছে। npm, pnpm, Yarn, Bun — এই সব manager ওই একই registry থেকেই package নামায়।

Package Manager ছাড়া কী সমস্যা হত?

Package Manager ছাড়া: → প্রতিটি library-র ওয়েবসাইটে গিয়ে manually ডাউনলোড করতে হত → প্রতিটি library-র dependencies-ও manually নামাতে হত → version mismatch হলে বুঝতে পারতে না → টিমের সবার কম্পিউটারে একই version রাখা কঠিন হত → "আমার কম্পিউটারে চলে, তোমার চলছে না কেন?" — এই সমস্যা! Package Manager থাকলে: npm install ← একটি কমান্ডে সব হয়ে যায় ✅
Package Manager তৈরি সাল বিশেষত্ব গতি
npm ২০১০ সবচেয়ে পুরনো, Node.js-এর সাথে আসে, বিশাল registry মাঝারি
pnpm ২০১৭ disk space বাঁচায়, hard links ব্যবহার, monorepo-বান্ধব দ্রুত
Yarn ২০১৬ Facebook তৈরি, reliable lockfile, workspaces দ্রুত
Bun ২০২২ All-in-one runtime+manager, Zig-এ লেখা, সবচেয়ে দ্রুত 🚀 সবচেয়ে দ্রুত

🔷 Package Manager কীভাবে কাজ করে?

সব package manager-এর কাজের ধাপ মূলত একই:

তুমি লেখো: npm install express Package Manager কী করে: ১. npmjs.com registry-তে "express" খোঁজে ২. সর্বশেষ stable version খুঁজে বের করে ③ express-এর সব dependencies-এর তালিকা নেয় ④ সব package ডাউনলোড করে ⑤ node_modules/ ফোল্ডারে রাখে ⑥ package.json-এ dependency যোগ করে ⑦ lockfile আপডেট করে (exact version record) ফলাফল: node_modules/ express/ body-parser/ ← express-এর dependency accepts/ ← আরো nested dependency ... package.json ← "express": "^4.18.2" package-lock.json ← exact version সংরক্ষিত

🟥 npm — Node Package Manager সম্পূর্ণ গাইড

npm হলো JavaScript-এর সবচেয়ে পুরনো ও জনপ্রিয় package manager। Node.js ইনস্টল করলেই npm আসে। ২০ লক্ষেরও বেশি package নিয়ে বিশ্বের বৃহত্তম software registry।

🔷 npm কী এবং কীভাবে তৈরি হলো?

npm (Node Package Manager) ২০১০ সালে Isaac Z. Schlueter তৈরি করেন। এটি একসাথে তিনটি জিনিস:

npm তিনটি জিনিস এক নামে: ১. Command-line tool → terminal-এ "npm install" টাইপ করো ২. Registry → npmjs.com-এ ২০ লক্ষ+ package জমা আছে ③. Website → npmjs.com-এ package খোঁজো, documentation পড়ো
সাল ঘটনা
২০১০ npm v1 — Node.js-এর সাথে প্রথম release।
২০১৪ npm v2 — nested node_modules, সমস্যা শুরু।
২০১৬ npm v3 — flat node_modules structure, Yarn-এর competition শুরু।
২০১৭ npm v5 — package-lock.json যোগ (Yarn-এর জবাব)।
২০২০ GitHub/Microsoft npm কিনে নেয় $7.5 billion-এ।
২০২১ npm v7 — Workspaces, peer deps auto-install।
২০২৩+ npm v10 — provenance statements, OIDC, আরো দ্রুত।

🔢 Versioning ও Semantic Versioning (SemVer)

🔷 SemVer — MAJOR.MINOR.PATCH বিস্তারিত

Format: MAJOR . MINOR . PATCH উদাহরণ: 4 . 18 . 2 MAJOR (4) → Incompatible API changes — পুরনো code ভাঙতে পারে MINOR (18) → New feature — backward compatible PATCH (2) → Bug fix — backward compatible Pre-release versions: 1.0.0-alpha.1 → initial alpha 1.0.0-beta.3 → beta testing 1.0.0-rc.1 → release candidate 1.0.0 → stable! Build metadata: 1.0.0+build.20240601

Version Range Notation বিস্তারিত

Notation অর্থ উদাহরণ range
"4.18.2" Exact version শুধু 4.18.2
"^4.18.2" MAJOR lock — compatible 4.18.2 ≤ x < 5.0.0
"~4.18.2" MINOR lock — approximately 4.18.2 ≤ x < 4.19.0
">=4.18.2" Greater or equal 4.18.2 এবং তার বেশি সব
">=4.18.2 <5" Range 4.18.2 থেকে 5 এর আগে
"4.18.x" Wildcard 4.18.0 থেকে 4.18.x
"4.x" Major wildcard 4.0.0 থেকে 4.x.x
"*" Any version সর্বশেষ (বিপজ্জনক!)
"latest" Latest tag সর্বশেষ stable
"1.2.3 || 2.0.0" OR condition 1.2.3 অথবা 2.0.0

🔒 package-lock.json — Deterministic Installs

🔷 package-lock.json কেন দরকার?

package.json-এ version range থাকে (^4.18.2), তাই আজ install করলে 4.18.2 আসতে পারে, কাল 4.19.0। package-lock.json exact version lock করে — পুরো টিমের এবং CI/CD-এর install হুবহু একই হবে।

package.json: "express": "^4.18.2" ← range (flexible) package-lock.json: "version": "4.18.2", ← exact lock! "integrity": "sha512-...", ← checksum verify "resolved": "https://..." ← exact URL
✅ সোনালী নিয়ম:
→ package-lock.json → সবসময় Git-এ commit করো
→ node_modules/ → সবসময় .gitignore-এ রাখো

🟦 pnpm — Performant npm

pnpm হলো npm-এর দ্রুততম ও সবচেয়ে disk-efficient বিকল্প। Content-addressable store দিয়ে disk space বাঁচায়।

🔷 pnpm কেন তৈরি হলো ও কী সুবিধা?

২০১৭ সালে Zoltan Kochan pnpm তৈরি করেন। npm-এর দুটো বড় সমস্যা ছিল: disk space অপচয় (প্রতিটি প্রজেক্টে একই package আলাদাভাবে থাকত) এবং ghost dependencies (package.json-এ নেই কিন্তু import করা যেত)।

npm-এর সমস্যা: project-a/node_modules/lodash@4.17.21 (5 MB) project-b/node_modules/lodash@4.17.21 (5 MB আরো!) project-c/node_modules/lodash@4.17.21 (5 MB আরো!) pnpm-এর সমাধান: ~/.pnpm-store/lodash@4.17.21 (5 MB — শুধু একবার!) project-a/node_modules/lodash → hard link ← 0 MB project-b/node_modules/lodash → hard link ← 0 MB project-c/node_modules/lodash → hard link ← 0 MB ✅ disk space ৯০% পর্যন্ত বাঁচতে পারে!
সুবিধা বিবরণ
Disk space বাঁচায় Global content-addressable store — একই package একবার download
দ্রুত install cache থেকে hard link — no copying
Strict node_modules package.json-এ না থাকলে import করা যায় না (ghost dependency নেই)
Monorepo-বান্ধব শক্তিশালী workspaces, filtering
npm compatible package.json, .npmrc সব কাজ করে
⚠️ Ghost Dependency কী?
npm-এ flat node_modules structure-এ তুমি এমন package import করতে পারো যা তোমার package.json-এ নেই কিন্তু কোনো dependency-র dependency হিসেবে install হয়েছে। এটি একটি বড় bug — production-এ সেই dependency update/remove হলে তোমার code ভাঙবে। pnpm এটি সম্পূর্ণ বন্ধ করে।

🔒 yarn.lock — Deterministic Lockfile

🔷 yarn.lock কেন npm-এর চেয়ে ভালো ছিল?

Yarn-ই প্রথম একটি human-readable, flat lockfile এনেছিল। npm-এর package-lock.json complex JSON হলেও yarn.lock পড়তে সহজ।

# yarn.lock ফাইলের উদাহরণ: express@^4.18.2: version "4.18.2" resolved "https://registry.yarnpkg.com/express/-/express-4.18.2.tgz#..." integrity sha512-... dependencies: accepts "~1.3.8" body-parser "1.20.1" ... # → মানুষ পড়তে পারে, diff করতে পারে, review করতে পারে
✅ yarn.lock সবসময় commit করো। এটি না করলে CI-তে ভিন্ন version install হতে পারে। শুধু node_modules/ .gitignore-এ রাখো।
